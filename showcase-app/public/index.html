Document API

Perform key-value CRUD operations on airport documents using Couchbase's REST Document API. Each operation maps directly to a single HTTP call.

Query API (SQL++)

Execute SQL++ (N1QL) queries against Couchbase collections. The Query API lets you join collections, filter, aggregate, and sort — all over HTTP with no SDK required.

/_p/query/query/service
POST Find Airport Routes

Query all flight routes (incoming and outgoing) for a given airport using its FAA or ICAO code.

SQL++ Statement
SELECT r.*
FROM `travel-sample`.`inventory`.route r
WHERE r.sourceairport = $code
   OR r.destinationairport = $code
ORDER BY r.sourceairport, r.destinationairport
LIMIT 10

POST Find Airlines at Airport

JOIN routes with airline documents to find all airlines operating at an airport — a cross-collection SQL++ JOIN.

SQL++ Statement (JOIN)
SELECT DISTINCT al.name, al.iata, al.icao, al.country
FROM `travel-sample`.`inventory`.route r
JOIN `travel-sample`.`inventory`.airline al
  ON r.airlineid = META(al).id
WHERE r.sourceairport = $code
   OR r.destinationairport = $code
ORDER BY al.name LIMIT 20

Full-Text Search API

Geo-spatial search powered by Couchbase FTS. Find hotels within a configurable radius of any airport — using the airport's coordinates fetched via the Document API, then passed to the FTS geo-distance query.

/_p/fts/api/bucket/{b}/scope/{s}/index/{i}/query
POST Find Hotels Near Airport

Two-step operation: first fetches airport coordinates via Document API, then runs a geo-distance FTS query against the hotel-geo-index.

1
Document API

GET airport doc → extract geo.lat, geo.lon

→
2
FTS API

POST geo-distance query → sorted hotel hits

FTS Query Shape
{
  "query": {
    "location": { "lat": <from airport>, "lon": <from airport> },
    "distance": "5km",
    "field": "geo"
  },
  "sort": [{ "by": "geo_distance", "field": "geo", "unit": "km" }],
  "fields": ["*"],
  "size": 20
}
Requires the hotel-geo-index FTS index. Run scripts/create-fts-index.js to create it.
